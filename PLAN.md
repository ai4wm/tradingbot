# 키움 REST API 조건검색·종목조회 화면 구현 계획

> 최종 업데이트: 2026-07-14. 초기 계획서가 아니라 현재 구조와 향후 확장 원칙을 설명하는 기준 문서다.

## 1. 목표와 현재 범위

영웅문 [0156] 조건검색실시간 및 [0198] 실시간 종목조회순위와 유사한 조회 도구를 PySide6로 구현한다. 키움 REST API와 웹소켓을 사용해 조건 편입/이탈, 실시간 시세, 예상체결, 순위 자료를 한 화면에서 판독한다.

현재 범위는 **조회와 알림**이다. 주문, 체결통보, 잔고 관리, 자동매매는 구현하지 않았다.

## 2. 기술 구조

- Python 3.11, PySide6, qasync, websockets, httpx, python-dotenv
- Qt 이벤트 루프와 asyncio를 qasync로 통합한다.
- GUI 갱신과 네트워크 처리는 같은 이벤트 루프에서 수행한다.
- 사용자 설정은 `QSettings` 기반 `layout.ini`, 진단 로그는 `bot.log`에 저장한다.
- API 키는 Git에서 제외한 `.env`에서 읽는다.

## 3. 파일별 책임

| 파일 | 책임 |
|---|---|
| `config.py` | 실전/모의 도메인, 키, 실시간 등록 한도, REST 속도, 틱 페이지 한도 |
| `api.py` | 토큰, REST TR, 종목 분류, 조회순위/NXT/거래량·대금 순위, 틱·분봉 조회 |
| `ws.py` | LOGIN/PING/재접속, 조건검색, 실시간 REG/REMOVE, 접미사별 참조수와 REAL 라우팅 |
| `gui.py` | 조건검색 표 모델, 정렬, 델리게이트, 체결 지표, 선택선, 체크 표시, 화면 설정 |
| `rank.py` | [0198] 전용 순위창, 주기 조회, 순위 선택 유지, 알림 판정과 소리 |
| `main.py` | 앱/창 조립, 최대 3개 뷰, 메뉴별 데이터 소스, 테마, IP·시계, 설정 복원 |
| `DESIGN.md` | 화면 표현과 상호작용 기준 |
| `NEXT.md` | 실행 방법, 현재 구현 현황, 다음 작업 결정 여부 |

## 4. 데이터 흐름

1. 시작 시 토큰을 확보하고 종목 기본 분류를 적재한다.
2. 웹소켓 로그인 후 조건식 목록을 받아 콤보박스에 채운다.
3. 조건 등록 결과의 초기 종목과 이후 편입/이탈을 해당 창으로 라우팅한다.
4. 여러 창이 같은 종목을 쓰면 실시간 등록 참조수를 공유한다.
5. REST 조회 메뉴는 일정 주기로 스냅샷을 받고, 필요한 종목만 실시간 시세에 등록한다.
6. REAL 수신값은 접미사(``, `_AL`, `_NX`)까지 확인해 올바른 창에 전달한다.
7. 모델은 원시값을 저장하고 화면용 파생값과 정렬 키를 별도로 제공한다.

## 5. 조회 모드

| 메뉴 | 데이터 소스 | 특징 |
|---|---|---|
| 일반 조건식 | 웹소켓 CNSRREQ | 초기 편입 목록 및 실시간 편입/이탈 |
| `[순위]조회순위` | `ka00198` | 30초/1분/10분/1시간/당일 기준 |
| `[NXT]등락률순위` | `ka10027`, `stex_tp=2` | NXT 전용, 상위 20개, `_NX` 시세 |
| `[급증]거래량급증` | 거래량 급증 TR | 설정 구간 대비 급증 종목 |
| `[대금]거래대금상위` | 거래대금 순위 TR | 거래대금 상위 종목 |

NXT 전용 모드는 전역 통합 체크와 분리한다. 일반 창의 통합 체크는 KRX 전용(`` 접미사)과 KRX+NXT 통합(`_AL`)을 전환한다.

## 6. 핵심 파생 지표

### 체결/분

- 최근 60초의 개별 체결 틱 개수다. 거래량이 아니라 체결 **건수**다.
- 최근 10초를 분당 속도로 환산해 이전 50초 속도와 비교한다.
- 최소 표본과 20% 완충 조건을 만족할 때만 증가/감소 삼각형을 표시한다.
- 값이 500/1000/1500을 넘으면 노랑/주황/빨강 단계 배경을 사용한다.

### 매수%

- 최근 3분의 공격적 매수 체결량 ÷ 전체 공격적 체결량이다.
- 웹소켓 FID 15의 부호로 매수(양수)와 매도(음수)를 구분한다.
- 숫자는 3분 비중이며, 삼각형은 최근 1분과 이전 2분의 비중 차이다.
- 단독 매매 신호가 아니라 순간 수급 확인용이다.

### 시총(억)

- 상장주식수 × 현재가를 억원 단위로 표시한다.
- 아직 체결가가 없으면 전일종가를 사용한다.

## 7. 예상체결·단일가·상한가

- 예상값 표시를 0D의 단순 변화만으로 켜지 않는다.
- 0H 수신, 단일가 대상 표시, VI 실시간(1h), 동시호가/VI REST 확인을 확정 신호로 사용한다.
- 예상가 0, VI 해제, 단일가가 아닌 상태에서 체결 재개 시 예상값을 해제한다.
- 실시간 송신이 없는 단일가 종목은 3초 REST 조회로 보완한다.
- 상한가 진입시간은 실시간 0B 체결시각을 우선하고 누락 시 `ka10079` 틱차트, 최대 6페이지 초과 시 분봉을 사용한다.

## 8. 안정성 규칙

- REG/REMOVE는 종목별 연속 전송하지 않고 짧게 모아 한 번에 전송한다.
- 등록 상태는 `(순수 종목코드, 명시 접미사)`로 관리한다.
- 실시간 등록은 95종목을 상한으로 둔다.
- 서버 PING 패킷은 즉시 되돌려 보내고, 연결 종료 시 최대 30초 지수 백오프로 재접속한다.
- 재접속 후 사용 중인 조건식과 실시간 종목을 재등록한다.
- 토큰·비밀키는 로그에 기록하지 않는다.
- REST는 설정된 속도 제한을 지키고 연속조회에는 `cont-yn`/`next-key`를 사용한다.

## 9. 화면 상태 보존

- 메인과 추가 창은 서로 다른 설정 접두사를 사용한다.
- 창 위치·크기, 열 너비, 정렬 열·방향, 선택 조건, 갱신 옵션을 저장한다.
- 순위형 화면은 갱신 후에도 선택한 순위 행을 복원한다.
- 선택 배경과 글자색은 바꾸지 않고 테마별 위·아래 선만 표시한다.
- 체크 표시는 창 활성 여부에 관계없이 같은 밝기로 보인다.

## 10. 검증 기준

- `py_compile`로 전체 Python 파일의 문법을 확인한다.
- `ws.py` 자가검증으로 파싱, 접미사 전환, 등록 참조수를 확인한다.
- `rank.py` 자가검증으로 모델 역할과 알림 판정을 확인한다.
- 장중에는 조건 편입/이탈, 체결 지표, VI/예상값, 상한가 시각을 영웅문과 대조한다.
- NXT 애프터마켓에는 `_NX`와 `_AL` 수신 및 NXT 순위를 확인한다.
- 네트워크 차단/복구와 장시간 PING 유지 시험을 수행한다.

## 11. 자동매매 확장 전 필수 조건

1. `IS_MOCK = True`와 모의투자 키로 전환한다.
2. 주문 TR과 주문 체결통보 실시간(00/04)을 별도 상태 기계로 구현한다.
3. 시작 시 잔고·미체결 조회로 상태를 복구한다.
4. 중복 주문, 최대 보유액, 일손실, 주문 속도 제한과 비상 정지를 먼저 정의한다.
5. 조회 화면의 순간 지표를 단독 주문 조건으로 사용하지 않고 분봉·호가·위험 조건과 함께 검증한다.
