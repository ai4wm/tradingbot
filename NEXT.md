## 핵심 동작 원리 (다시 안 헤매게)

- **예상값(exp) 표시 ON은 확정신호(hot)로만**: 0H 수신 / 단일가 마킹 / VI(1h) / 동시호가·VI REST.
  0D 23/24는 연속매매 중에도 미세하게 변하며 옴 → ON 신호로 쓰면 오탐(012160 사건).
  켜진 뒤 0D값 갱신은 허용. OFF: 예상가0 / 체결재개(거래량↑, 단일가 제외) / VI해제.
- **단일가 종목은 WS 무송신(실측 0건)** → 3초 REST 폴이 유일한 채널. 마킹 = orderWarning 2/3
  (정리매매/단기과열) + **50만주 미만 우선주**(저유동성 상시 단일가). 4=투자위험은 단일가 아님(실측).
- **REG/REMOVE는 반드시 묶음 1건** — 종목당 연사하면 서버 105110으로 뒷부분 조용히 거부
  (하루종일 "어떤 종목만 데이터 안 옴"의 진범이었음). App.queue_real(0.3초 모음)+ws 참조수.
- **상한가진입시각 = ka10079 틱차트**(초단위, 영웅문과 동일값 검증). 헤더 연속조회(cont-yn/next-key)
  최대 6페이지, 초과 시 분봉 폴백. 거래량 적은 순(점상 먼저) 순차 조회. 상한 유지중 캐시.
- **상한가정렬 재정렬은 스로틀**(200ms, 디바운스면 틱 폭주 때 영영 안 돎). 비그룹은 유효등락률
  (exp 살아있으면 예상등락률) 내림차순.
- **NXT**: 실시간은 코드에 `_AL`(통합)/`_NX` 접미사 붙여 REG하면 장후에도 수신(실측). 단
  WS 조건검색 stex_tp는 "K"만 허용 → NXT 편입/이탈은 이 API로 불가. 현재 미적용(KRX 전용).
- 종목명 셀 표식: 좌상단 노랑▲=NXT, 우상단 녹색▲=미수가능, 이름색 보라=코스닥/주황=관리종목.
  전부 ka10099 1회 로딩(MarketInfo). 순위 세로헤더 = 정렬 순위(1..N).
